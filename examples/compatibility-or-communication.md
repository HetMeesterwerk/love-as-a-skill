# Example: Compatibility or Communication?

## Purpose

This example shows how the `Love as a Skill` agent skill should help a user think clearly about whether a relationship problem is mainly a trainable communication issue or a deeper compatibility issue.

The goal is not to push the user toward "break up" or "just communicate better". The goal is to create a structured decision-support process.

The assistant should help the user:
- separate fixable skills from structural mismatches
- identify recurring patterns
- clarify needs, values, and boundaries
- avoid impulsive conclusions
- design a repair experiment
- set a decision checkpoint

## User prompt

```text
I keep wondering whether my relationship problems come from poor communication or from deeper incompatibility. Help me think this through clearly without jumping to either "break up" or "just communicate better."
```

## Skill trigger

This should trigger the skill because the user is asking for help with:
- relationship evaluation
- communication patterns
- compatibility
- values and needs
- uncertainty
- decision support
- avoiding simplistic advice

## What the assistant should do

The assistant should use the `Love as a Skill` framework to produce a structured response with these parts:

1. Define the distinction
2. Map the recurring problem
3. Use the five-layer model
4. Separate skill issues from compatibility issues
5. Create a repair experiment
6. Set a decision checkpoint
7. Add safety and quality checks

## Required response structure

```text
1. Core distinction
2. Five-layer analysis
   - Facts
   - Feelings
   - Meaning
   - Pattern
   - Skill
3. Skill problem indicators
4. Compatibility problem indicators
5. Repair experiment
6. Decision checkpoint
7. Questions to answer
8. What not to do
```

## Fully worked example output

### 1. Core distinction

A communication problem means:

```text
The relationship may improve if both people practice better timing, listening, emotional regulation, repair, and clearer requests.
```

A compatibility problem means:

```text
The relationship may remain painful even with better communication because the underlying needs, values, life goals, or relational expectations do not fit.
```

A useful question is not:

```text
Should we break up or try harder?
```

A better question is:

```text
Is this a trainable pattern both people are willing and able to work on, or a recurring mismatch that asks one or both people to become fundamentally different?
```

### 2. Five-layer analysis

#### Facts

Start with observable data.

```text
- What keeps happening?
- How often does it happen?
- What topic does it usually involve?
- What happens after the conflict?
- Does repair happen?
- Do both people acknowledge the pattern?
```

Example fact statements:

```text
We argue about time together almost every week.
When I raise the issue, my partner often withdraws.
Afterward, we usually reconnect, but the same issue returns.
```

Avoid:

```text
They clearly do not care.
We are doomed.
I am too needy.
They are incapable of love.
```

#### Feelings

Name the emotional cost.

```text
- Do I feel lonely, anxious, resentful, unseen, pressured, trapped, or constantly inadequate?
- Do I feel more like myself in this relationship, or less?
- Do I feel safe being honest?
```

Feelings are not final proof, but they are important data.

#### Meaning

Identify what the pattern seems to mean.

```text
Possible meanings:
- "My needs are too much."
- "They do not prioritize me."
- "Love means I should accept this."
- "If this were right, it would feel easier."
- "Maybe we are good people but not a good fit."
```

The skill should help the user test these meanings instead of jumping to conclusions.

#### Pattern

Map the repeated loop.

```text
1. A need or difference appears.
2. One person raises it.
3. The other person defends, withdraws, minimizes, or counterattacks.
4. The issue becomes about tone, blame, or pressure.
5. Temporary peace returns.
6. The original issue remains unresolved.
```

A repeated loop is more important than one bad conversation.

#### Skill

The trainable skill may be:

```text
- asking more clearly
- listening without defending
- repairing faster
- tolerating difference
- making explicit agreements
- naming needs without blame
- setting boundaries earlier
```

But the skill only matters if both people are willing to practice it.

### 3. Skill problem indicators

The issue is more likely to be a skill problem if:

```text
[ ] Both people can eventually understand the other's perspective.
[ ] Both people show remorse when they hurt each other.
[ ] Both people can name their part in the pattern.
[ ] The issue improves when discussed calmly.
[ ] The conflict is often about delivery, timing, assumptions, or defensiveness.
[ ] There is goodwill, but poor execution.
[ ] Repair attempts usually work.
[ ] Small behavioral changes create noticeable improvement.
```

Examples:

```text
- One person needs more reassurance, the other needs help not feeling accused.
- Both want closeness, but one pursues and the other withdraws under stress.
- Arguments escalate because both people defend too quickly.
- The same need could be met with clearer agreements.
```

### 4. Compatibility problem indicators

The issue is more likely to be a compatibility problem if:

```text
[ ] Core life goals conflict.
[ ] One person wants a type of relationship the other does not want.
[ ] Emotional availability needs are persistently mismatched.
[ ] Sexual expectations are persistently mismatched and not discussable.
[ ] Trust has been repeatedly broken without repair.
[ ] One person must shrink or perform to keep the peace.
[ ] The relationship requires one person to abandon a core value.
[ ] The same issue remains unchanged after honest attempts to repair.
[ ] There is contempt, chronic disrespect, fear, or coercion.
```

Examples:

```text
- One person wants children and the other clearly does not.
- One person wants monogamy and the other does not.
- One person needs emotional openness and the other refuses emotional conversations entirely.
- One person repeatedly violates agreed boundaries.
- One person's lifestyle, values, or availability makes the other's basic needs chronically unmet.
```

### 5. The repair experiment

Before making a final decision, test whether the problem is trainable.

Run a 2 to 4 week repair experiment.

#### Step 1: Define one recurring issue

```text
The recurring issue is:
We argue about how much quality time we need.
```

#### Step 2: Define one concrete request

```text
Request:
Can we plan two intentional evenings together per week without phones for the first hour?
```

#### Step 3: Define both people's responsibility

```text
My part:
I will ask directly instead of hinting or testing.

Their part:
They will respond directly instead of dismissing the need as pressure.
```

#### Step 4: Define the repair rule

```text
If either of us becomes defensive, we pause for 20 minutes and return to the conversation.
```

#### Step 5: Define the checkpoint

```text
After 3 weeks, we ask:
Did the pattern improve?
Did both people try?
Did repair happen faster?
Do I feel more respected and more myself?
```

### 6. Decision checkpoint

Use this checkpoint after the repair experiment.

```text
Green signs:
- Both people tried.
- The issue became less intense or less frequent.
- Repair became easier.
- Both people could name their own part.
- The relationship felt safer and clearer.

Yellow signs:
- There was some effort, but it was inconsistent.
- The issue improved only when one person carried all the emotional labor.
- Promises were made but not translated into behavior.

Red signs:
- The issue did not change.
- One person refused to engage.
- One person mocked, punished, threatened, or minimized the other's needs.
- The relationship requires self-abandonment.
- The user feels chronically smaller, unsafe, or emotionally alone.
```

### 7. Key questions

```text
What keeps repeating?
Have we both genuinely tried to repair it?
Is the issue about behavior, values, timing, trust, or life direction?
Do I feel more like myself in this relationship, or less?
Are we asking each other to become fundamentally different people?
Can we discuss hard things without punishment or collapse?
Is there enough goodwill to practice better skills?
If nothing changed for one year, would I still choose this?
```

### 8. What not to do

Avoid:

```text
- deciding based on one bad week
- ignoring a repeated pattern because there are good moments
- treating all conflict as a communication issue
- treating all discomfort as incompatibility
- using compatibility language to avoid accountability
- using "relationships take work" to excuse chronic disrespect
- outsourcing the decision entirely to an AI assistant
```

## Safety boundaries

The assistant should not:
- tell the user to break up based on limited information
- tell the user to stay because relationships require effort
- minimize dealbreakers
- diagnose either partner
- frame chronic fear, coercion, or humiliation as a normal communication issue
- make the decision for the user

If the user describes threats, violence, coercion, stalking, sexual pressure, financial control, or serious fear, the assistant should shift away from ordinary compatibility analysis and toward safety-oriented support.

## Quality criteria

A good answer should:
- reduce impulsive decision-making
- clarify what can and cannot be trained
- protect the user's values and boundaries
- avoid simplistic breakup or stay advice
- create a concrete repair experiment
- set a decision checkpoint
- help the user distinguish poor skills from poor fit
